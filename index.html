關於這份申請網頁，分享一個開發時的技術心得。

在過去，我習慣直接使用 @media 查詢來因應不同螢幕尺寸。
後來，我學習了使用 @mixin 封裝 media query，讓程式碼更易於維護。
在開發「榮總健康中心後台」時，我使用了 Bootstrap 內建的 breakpoint 功能。

這次申請網頁，我希望能讓程式碼更乾淨，因此嘗試了參照 Bootstrap 的寫法，以及參考了影片 Media query with SASS @mixin 的寫法。然而，我遇到了瓶頸，程式碼在終端機上沒有錯誤，卻無法正常運作。

卡關點

$breakpoints: (
  "xs": "max-width:575.98px",
  "sm": "(min-width:576px) and (max-width:767.98px)",
  "md": "(min-width:768px) and (max-width:991.98px)",
  "lg": "(min-width:992px) and (max-width:1199.98px)",
  "xl": "(min-width:1200px) and (max-width:1399.98px)",
  "xxl": "min-width:1400px",
);

@mixin breakpoint($user-value) {
  $map: map-get($breakpoints, $user-value);
    @if $map {
  @media (#{$map}) {
    @content;
  }
} @else {
   @error "Opps";
  }
}

我希望能使用 map-get 函數來取得下一個斷點的名稱，並將其作為 max-width的值。然而，程式碼在終端機上沒有錯誤，卻沒有產生預期的效果。

  1. $breakpoints定義一個變數來儲存所有斷點的名稱和值。
  2. @mixin 定義用於接收一個斷點名稱作為參數，參數名稱宣告它為$user-value,。
  3. 使用 map-get 函數根據斷點名稱獲取其值。
  4. 如果找到相應的值，則使用 @media 查詢來生成相應的 CSS 規則。
  5. 如果找不到相應的值，則拋出錯誤。
