@use 應用心得：如何讓 SCSS 程式碼更乾淨

在之前的工作中，由於沒有命名規則，所有 CSS 都為公開，導致以下問題：

  • 類別容易被不知道哪裡的 CSS 影響
  • 類別容易被父元素的 class style 影響
  • 無法隨便亂改

這件事讓我更加了解到私有變數的重要性，寫出讓人好維持程式碼非常的重要。因此，在這次申請網頁時，我嘗試使用 @use 替代 @import。
一開始，我還沒有搞清楚 @use 的規則，將其理解為作用域的意思。
因此，我在 all.scss 檔案中寫了以下程式碼：
@use "breakpoint" as *;

我以為這樣可以直接在 _grid.scss 檔案中引入 _breakpoint.scss 檔案的變數。但結果終端機出現了以下錯誤訊息：
Error: Undefined mixin.

我使用這個錯誤訊息去查詢，出現了許多不同解答方向。我沒辦法在 SCSS 順利的除錯，有時候單純看錯誤訊息不知道從何下手。
之後，我嘗試了以下方法：

  • 在 _grid.scss 檔案中使用 @import "breakpoint.scss"; 檔案，這樣引入是成功。
  • 在 all.scss 檔案中使用 @import 的方法，但我其他想要@use，@use需要在其他的樣式規則上。

經過這些嘗試，我認為我的問題主要出在對 @use 的應用理解不夠深入。我對 @use 的應用還很淺薄，甚至誤解了 as 的含義。as 其實是命名空間，而 @use 引用的是私有變數。雖然允許多個引用，但翻譯時只會執行一次。

為了深入了解 @use 的應用，我參考了一些 Sass 官方網站、筆記和影片。通過學習，我終於掌握了 @use 的正確用法。

使用 map-get 函數在 SCSS 中實現響應式佈局

在過去，我習慣直接使用 @media 查詢來因應不同螢幕尺寸。
後來，我學習了使用 @mixin 封裝 media query，讓程式碼更易於維護。
在開發「榮總健康中心後台」時，我使用了 Bootstrap 內建的 breakpoint 功能。

這次申請網頁，我希望能讓程式碼更乾淨，因此嘗試了參照 Bootstrap 的寫法，以及參考了影片 Media query with SASS @mixin 的寫法。然而，我遇到了瓶頸，程式碼在終端機上沒有錯誤，卻無法正常運作。

卡關點

$breakpoints: (
  "xs": "max-width:575.98px",
  "sm": "(min-width:576px) and (max-width:767.98px)",
  "md": "(min-width:768px) and (max-width:991.98px)",
  "lg": "(min-width:992px) and (max-width:1199.98px)",
  "xl": "(min-width:1200px) and (max-width:1399.98px)",
  "xxl": "min-width:1400px",
);

@mixin breakpoint($user-value) {
  $map: map-get($breakpoints, $user-value);
    @if $map {
  @media (#{$map}) {
    @content;
  }
} @else {
   @error "Opps";
  }
}

我希望能使用 map-get 函數來取得下一個斷點的名稱，並將其作為 max-width的值。然而，程式碼在終端機上沒有錯誤，卻沒有產生預期的效果。

  1. $breakpoints定義一個變數來儲存所有斷點的名稱和值。
  2. @mixin 定義用於接收一個斷點名稱作為參數，參數名稱宣告它為$user-value,。
  3. 使用 map-get 函數根據斷點名稱獲取其值。
  4. 如果找到相應的值，則使用 @media 查詢來生成相應的 CSS 規則。
  5. 如果找不到相應的值，則拋出錯誤。
